# California Source Catalog

CalDS now treats California-wide investigation coverage as a source-planning problem, not as a one-off scraper. Each `InvestigationProfile` receives a deterministic catalog of official statewide sources, generic county source families, and jurisdiction-specific overlays when the profile identifies a known local portal such as San Francisco.

The catalog does not claim that every record has been collected. It tells the workflow what must be checked, what can be accessed through public web sources, and what remains a records-request or credentialed-access blocker.

## Statewide Connectors

- Internal Revenue Service tax-exempt organization search, raw Form 990 XML, and return images: https://www.irs.gov/charities-non-profits/search-for-tax-exempt-organizations
- ProPublica Nonprofit Explorer API summaries tied to IRS filings: https://projects.propublica.org/nonprofits/api
- Federal Audit Clearinghouse public audit API and reporting packages: https://www.fac.gov/api
- California Attorney General Registry of Charities and Fundraisers: https://oag.ca.gov/charities
- California Secretary of State business search and filing images: https://bizfileonline.sos.ca.gov/search/business
- California State Auditor reports: https://www.auditor.ca.gov/reports/
- California Grants Portal and open-data grant records: https://www.grants.ca.gov/
- Open FI$Cal expenditure and vendor transaction reports: https://open.fiscal.ca.gov/
- California Homeless Data Integration System: https://bcsh.ca.gov/calich/hdis.html
- California Secretary of State CAL-ACCESS and Power Search: https://powersearch.sos.ca.gov/
- California Fair Political Practices Commission campaign, lobbying, and ethics sources: https://www.fppc.ca.gov/
- Federal PACER case locator: https://pacer.uscourts.gov/

## County and Local Connectors

For all California profiles, the catalog adds county contract, payment, board-file, and provider-outcome source families. These are intentionally marked `public_or_records_request` because there is no single portal that covers all California counties with the same schema. A county source family is complete only when the run has one of these states:

- a citation-ready public record hit,
- a documented no-public-record search with enough detail to reproduce the search,
- a hard external blocker such as credentialed access, fees, rate limits, missing public portal coverage, or a public-records request requirement.

San Francisco profiles add the current official/public overlays:

- San Francisco Open Data payment rows: https://data.sfgov.org/resource/qkex-vh98.json
- San Francisco Legislative Research Center and board files: https://www.sf.gov/legislative-research-center-lrc
- San Francisco Ethics Commission lobbying disclosure data: https://sfethics.org/disclosures/lobbyist-disclosure/lobbyist-disclosure-data
- San Francisco Controller reports: https://www.sf.gov/departments/city-administrator/office-controller
- San Francisco HSH data and archived reports: https://hsh.archive.sf.gov/
- San Francisco HSH department records: https://www.sf.gov/departments/homelessness-and-supportive-housing

## How This Changes Runs

`SourceAcquisitionPlannerService` now loads this catalog for every profile. If the corpus does not contain a required source family, the source-acquisition plan lists the connector IDs, required artifacts, official source URLs, retryability, and access blockers. Missing county records are no longer anonymous gaps.

Use the CLI to preview coverage before a full run:

```powershell
python -m calds_runtime source-catalog --jurisdiction "San Francisco, California" --profile-file data\investigation_profiles\sf_homelessness.json --output-file runs\source-catalog\sf_catalog.json
python -m calds_runtime source-catalog --jurisdiction "California" --profile-file data\investigation_profiles\ca_statewide_homelessness.json --output-file runs\source-catalog\ca_statewide_catalog.json
```

Use the normal source-plan preview to see entity-specific blockers:

```powershell
python -m calds_runtime plan-source-acquisition --case-file <case.json> --corpus-dir <corpus-dir> --output-file runs\source-catalog\source_plan_preview.json
```

## Current Boundary

This makes the generic spine available for all California investigations at the source-family and connector-planning layer. It does not mean CalDS has completed bespoke scrapers for all 58 counties, every superior court portal, every county contract system, or every provider-specific outcome store. Those remain explicit repair tasks generated by the completeness controller and source-acquisition plan.
